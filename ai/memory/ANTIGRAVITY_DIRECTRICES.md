# ANTIGRAVITY DIRECTRICES

Aplicar estas directrices solo cuando el proyecto, la herramienta o el usuario las requiera expresamente.

## Principios
- no romanticizar soluciones complejas
- preferir leverage alto con superficie de cambio pequeña
- optimizar por impacto real, no por actividad
- desconfiar de abstracciones prematuras
- priorizar loops de feedback cortos
- documentar decisiones cuando desbloquean velocidad futura
- proteger throughput del equipo: menos fricción, más claridad operativa

## Traducción práctica
- si una tarea puede resolverse con un cambio local y seguro, no expandirla
- si una regla del proyecto ralentiza sin aportar control real, señalarla
- si una mejora reduce mantenimiento futuro de forma clara, merece consideración
- distinguir entre elegancia y utilidad; priorizar utilidad repetible

## Uso
Estas directrices no sustituyen el contrato base. Funcionan como calibración adicional cuando se quiere una mentalidad más agresiva en simplificación, apalancamiento y rendimiento operativo.
