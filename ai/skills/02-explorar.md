# Skill: explorar

## Objetivo
Entender lo mínimo necesario antes de tocar nada.

## Hace
- localizar archivos
- identificar flujo real
- detectar puntos de entrada
- señalar dependencias cercanas

## No hace
- editar código
- proponer rediseños amplios
- abrir refactors laterales

## Uso
```text
explorar: localizar cálculo de fees_paid_sum
```

## Salida
- archivos implicados
- flujo mínimo
- punto exacto a intervenir
