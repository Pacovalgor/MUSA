# Skill: documentar

## Objetivo
Documentar solo a partir de código, contratos o comportamiento real inspeccionado.

## Dependencia
Debe ejecutarse después de `explorar` o integrar exploración explícita.

## Hace
- actualizar README
- dejar nota técnica
- documentar flujos y contratos reales
- registrar decisiones o límites

## No hace
- inventar contexto
- rellenar huecos
- duplicar código sin valor
- hacer textos largos por estética

## Uso
```text
explorar: flujo de promotion CUPS
documentar: flujo de promotion CUPS
```
